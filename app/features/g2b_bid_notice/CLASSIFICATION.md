# 개인 수집 설정 분류 규칙

이 기능은 사용자 계정 하나의 개인 수집 설정을 기준으로 동작한다. 여러 설정은 사용자가 서로 다른 사업을 분리할 때만 추가하며, 설정별 조건·수신자·Sheets 대상·결과는 독립적이다.

## 컬럼별 판정

| 조건 | 판정 원본 | 처리 |
| --- | --- | --- |
| 대상 기관 그룹 | 수요기관명, 기관코드, 별칭, 산하기관 목록 | 공고명과 합치지 않고 목록의 정확한 일치만 확인 |
| 업무구분·조달 구분 | 나라장터 메타데이터 | 키워드 검색 없이 원본 값과 정확히 비교 |
| 필수·제외 키워드 | `bidNtceNm` | 필수는 모두 포함, 제외는 하나라도 포함 시 불일치 |
| 기초금액 | `base_amount` | 추정가격·배정예산으로 대체하지 않음 |
| 참가 가능 지역 | 지역제한 상세 원본 | 미확인 값을 전국으로 추정하지 않음 |
| 게시일자 | `bidNtceDt` | 나라장터 목록 조회 기간과 동일한 게시일자 기준으로 비교 |

모든 활성 조건은 AND 관계다. 원본값이 명확하게 불일치하면 `EXCLUDE`, 활성 상세 조건의 원본값이 없으면 `REVIEW`, 모든 활성 조건이 확인되고 일치하면 `PRIORITY`가 된다.

## 수집 단계

1. 목록 API에서는 업무구분·공고명·기간처럼 API가 직접 지원하는 조건만 요청한다. 공고명 최종 판정은 언제나 목록 응답의 `bidNtceNm`만으로 수행한다.
2. 기초금액, 지역, 제안서 마감일, 2단계 입찰 여부는 목록 결과가 불완전하면 상세 API 보강 대상으로 남긴다. 현재 feature는 값을 추정하지 않고 `DETAIL_REQUIRED` 상태로 돌려준다.

공용 DB 모델과 마이그레이션은 이 feature에서 만들거나 수정하지 않는다. 분류가 끝난 입찰공고는 [CONTRACT.md](CONTRACT.md)의 공통 저장 계약으로 통합 저장 함수에 전달한다.
